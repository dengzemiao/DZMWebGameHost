# 中国象棋游戏架构

## 一、概述

中国象棋作为游戏插件运行在 DZMWebGameHost 平台上，遵循 GameAdapter 标准接口。全部前端实现（包括 AI），服务端仅负责房间管理、消息转发、状态缓存。

**文件结构**：
```
games/chess/
├── game.js    # ChessGameAdapter（游戏适配器）+ ChessRules（规则引擎）
├── ai.js      # ChessAI（Minimax + Alpha-Beta 剪枝）
└── game.css   # 棋盘 + 棋子 + 动画样式
```

### 2.3 标准接口方法

| 方法 | 触发时机 | 作用 |
|------|---------|------|
| `init()` | 游戏脚本加载后 | 创建初始棋盘、构建 DOM、首次渲染 |

## 三、核心流程详解

### 3.1 _canControl() 判断逻辑

```js
_canControl() {
  // 观战者不可操作
  if (this.isSpectator) return false;
  
  // 以下任一条件返回 false：
  // 1. 游戏已结束
  // 2. PvP 模式下游戏未开始
  // 3. 不是自己的回合
  return !(
    this.gameOver || 
    (!this.gameStarted && this.roomType !== 'pv_ai') || 
    this.currentTurn !== this.role
  );
}
```

**注意**：PvAI 模式跳过 gameStarted 检查，创建房间即可走棋。

### 3.2 点击事件流程

```
用户点击
├── 点击棋子 (.chess-piece) → _onPieceClick(r, c)
│   ├── !_canControl()
│   │   └── 点击己方棋子 → 显示原因提示
│   │       ├── gameOver → "游戏已结束"
│   │       ├── !gameStarted (PvP) → "等待对手加入..."
│   │       └── currentTurn !== role → "等待对方走棋"
│   │
│   ├── 已选中状态
│   │   ├── 点击合法目标 → _doMove() → 走棋
│   │   ├── 点击己方棋子 → 切换选中
│   │   └── 点击无效位置 → 取消选中
│   │
│   └── 未选中状态
│       └── 点击己方棋子 → 选中，显示落点提示
│
├── 点击落点提示 (.move-hint) → _onHintClick(tr, tc)
│   └── 有选中棋子 → _doMove()
│
└── 点击棋盘空白 (canvas) → _onBoardClick(e)
    └── 有选中棋子
        ├── 计算点击坐标 → 判断是否合法目标
        ├── 合法 → _doMove()
        └── 不合法 / 超出范围 → 取消选中
```

### 3.3 走棋流程

**本地走棋 (_doMove)**：
```
_doMove(fr, fc, tr, tc)
  │
  ├── _applyMove(fr, fc, tr, tc)
  │   ├── 移动棋子：board[tr][tc] = board[fr][fc]，board[fr][fc] = null
  │   ├── 记录 lastMove
  │   ├── moveCount++
  │   ├── 切换 currentTurn
  │   ├── 检测将军 → 更新 checkColor
  │   └── 检测将死（getAllMoves 为空）
  │       └── 是 → gameOver=true → 300ms 后 notifyGameOver()
  │
  ├── 清除选中，重新渲染
  │
  ├── PvP 模式
  │   └── sendAction({ action:'move', fr, fc, tr, tc, game_state: snapshot })
  │
  ├── PvAI 模式
  │   └── sendAction({ action:'sync_state', game_state: snapshot })
  │
  └── PvAI 模式 + 游戏未结束
      └── _triggerAI()
```

**AI 走棋 (_triggerAI)**：
```
_triggerAI()
  │
  ├── 检查是否 AI 回合（currentTurn !== role）
  ├── 检查游戏是否结束
  │
  ├── 显示 "AI 思考中..." 动画
  │
  ├── 延迟执行（easy:300ms, normal:500ms, hard:800ms）
  │   │
  │   ├── 调用 ChessAI.bestMove(board, currentTurn, difficulty, ChessRules)
  │   │
  │   ├── 隐藏思考动画
  │   │
  │   └── 有走法 + 游戏未结束
  │       ├── _applyMove(mv.fr, mv.fc, mv.tr, mv.tc)
  │       ├── _render()
  │       └── sendAction({ action:'sync_state', game_state: snapshot })
```

**远程走棋 (onRemoteAction)**：
```
onRemoteAction(data)
  │
  ├── data.action === 'move'
  │   └── _applyMove(fr, fc, tr, tc) → _render()
  │
  └── data.action === 'sync_state' && data.game_state
      └── restoreGameState(data.game_state)
```

### 3.4 状态同步机制

**状态快照格式 (getStateSnapshot)**：
```js
{
  board: [                    // 10×9 棋盘数组
    [{ type:'R', color:'black' }, null, ...],
    ...
  ],
  currentTurn: 'red',         // 当前走棋方
  moveCount: 10,              // 总步数
  lastMove: {                 // 上一步记录
    fr: 0, fc: 0,             // 起点
    tr: 2, tc: 0              // 终点
  },
  checkColor: null,           // 被将军方（或 null）
  gameOver: false,            // 是否结束
  winner_role: 'red'          // 胜方（仅 gameOver 时）
}
```

**同步场景**：

| 场景 | 触发方 | 消息内容 | 接收方处理 |
|------|--------|---------|-----------|
| PvP 走棋 | 走棋方 | `game_action { action:'move', ..., game_state }` | 对手 _applyMove，观战者 onRemoteAction |
| PvAI 走棋 | 玩家/AI | `game_action { action:'sync_state', game_state }` | 观战者 restoreGameState |
| 新人加入 | 服务端 | `room_joined { room: { game_state } }` | room.js 调用 restoreGameState |
| 断线重连 | 服务端 | `room_joined { room: { game_state }, reconnected }` | 同上 |
| 观战中途加入 | 服务端 | `game_action { action:'sync_state', game_state }` | restoreGameState |

**服务端状态缓存**：
- `lobby.record_game_state(room_id, state)` - 每次 game_action 带 game_state 时调用
- `lobby.get_room_game_state(room_id)` - 加入/重连时获取
- `lobby.clear_room_game_state(room_id)` - 再来一局/房间销毁时清空

## 四、游戏生命周期

### 4.1 PvP 模式

```
创建房间
  → room_created
  → init() → 渲染初始棋盘
  → gameStarted = false
  → 等待对手（状态栏显示"等待中"）

对手加入
  → room_updated(state: playing)
  → game_restart 消息
  → onGameStart() → gameStarted = true
  → 红方可走棋

正常对局
  → 轮流走棋
  → 每步带 game_state 上报服务端

一方离开
  → room_updated(state: waiting)
  → game_over(reason: opponent_leave)
  → onOpponentLeave() → 显示通知
  → 席位保留，等待重连或新玩家

再来一局
  → 点击"再来一局"按钮
  → ws.send({ type: 'restart_game' })
  → 服务端广播 game_restart
  → reset() → 恢复初始棋盘
  → onGameStart() → 游戏开始
```

### 4.2 PvAI 模式

```
创建房间
  → room_created
  → init() → 渲染初始棋盘
  → game_restart 消息
  → onGameStart()
    → gameStarted = true
    → 若玩家执黑（AI 红方先手）→ _triggerAI()

正常对局
  → 玩家走棋 → _triggerAI() → AI 走棋
  → 每步 sync_state 供观战者同步

观战者加入
  → room_joined(game_state)
  → restoreGameState() → 恢复当前棋盘
  → 观战者只能看，不能操作
```

### 4.3 观战模式

```
以观战身份加入
  → room_joined(game_state?)
  → initGameAdapter(config.isSpectator = true)
  → 若有 game_state → restoreGameState()

游戏进行中
  → 收到 game_action
    ├── action:'move' → onRemoteAction() → _applyMove()
    └── action:'sync_state' → restoreGameState()
  → 棋盘实时更新，但 _canControl() 返回 false

游戏结束
  → 收到 game_over
  → 聊天区显示结果（不弹窗）

切换参战
  → ws.send({ type: 'switch_role', to_spectator: false })
  → room_updated → 更新 isSpectator/role
  → 重新渲染，恢复操作能力
```

## 五、ChessRules 规则引擎

### 5.1 概述

纯函数规则模块，无状态、无副作用，供 Adapter 和 AI 共用。

### 5.2 API

```js
const ChessRules = {
  getCandidates(board, r, c),  // 获取候选移动（不考虑自将）
  getValidMoves(board, r, c),  // 获取合法走法（排除自将）
  getAllMoves(board, color),   // 获取某方所有合法走法
  isInCheck(board, color),     // 判断某方是否被将军
};
```

### 5.3 棋子类型与走法

| 类型 | 红方 | 黑方 | 走法规则 |
|------|------|------|---------|
| K | 帅 | 将 | 九宫内一格（上下左右）+ 将帅对脸（同列无阻挡可直接吃） |
| A | 仕 | 士 | 九宫内斜走一格 |
| B | 相 | 象 | 田字走（斜走两格），不能过河，象眼不能被堵 |
| N | 馬 | 马 | 日字走（先直后斜），马腿不能被堵 |
| R | 車 | 車 | 直线走（横竖任意格），不能跨越棋子 |
| C | 炮 | 炮 | 直线走，吃子必须隔一子（炮架） |
| P | 兵 | 卒 | 前进一步，过河后可横走一步 |

### 5.4 九宫格坐标

```
红方九宫（行 7-9，列 3-5）：
[7,3] [7,4] [7,5]
[8,3] [8,4] [8,5]
[9,3] [9,4] [9,5]

黑方九宫（行 0-2，列 3-5）：
[0,3] [0,4] [0,5]
[1,3] [1,4] [1,5]
[2,3] [2,4] [2,5]

士的合法位置（九宫的四角+中心）：
红方：[7,3] [7,5] [8,4] [9,3] [9,5]
黑方：[0,3] [0,5] [1,4] [2,3] [2,5]
```

### 5.5 过河判断

```js
// 红方视角：行 0-4 为对方阵营（过河），行 5-9 为己方
// 黑方视角：行 5-9 为对方阵营（过河），行 0-4 为己方

// 象不能过河
const riverLimit = isRed ? 5 : 4;
if (isRed && nr < riverLimit) continue;   // 红方象不能到行 0-4
if (!isRed && nr >= riverLimit) continue; // 黑方象不能到行 5-9

// 兵过河判断
const crossed = isRed ? r < 5 : r >= 5;
```

### 5.6 将军检测

```js
function isInCheck(board, color) {
  // 1. 找到将/帅位置
  let kr = -1, kc = -1;
  for (let r = 0; r < 10; r++) {
    for (let c = 0; c < 9; c++) {
      const p = board[r][c];
      if (p && p.type === 'K' && p.color === color) {
        kr = r; kc = c; break;
      }
    }
  }
  if (kr < 0) return true; // 将已被吃

  // 2. 检查对方所有棋子是否能攻击到将的位置
  const opp = color === 'red' ? 'black' : 'red';
  for (let r = 0; r < 10; r++) {
    for (let c = 0; c < 9; c++) {
      const p = board[r][c];
      if (!p || p.color !== opp) continue;
      const cands = getCandidates(board, r, c);
      if (cands.some(([mr, mc]) => mr === kr && mc === kc)) {
        return true;
      }
    }
  }
  return false;
}
```

### 5.7 合法走法过滤

```js
function getValidMoves(board, r, c) {
  const piece = board[r][c];
  if (!piece) return [];
  
  const color = piece.color;
  const candidates = getCandidates(board, r, c);
  const valid = [];
  
  for (const [tr, tc] of candidates) {
    // 模拟走棋
    const captured = board[tr][tc];
    board[tr][tc] = piece;
    board[r][c] = null;
    
    // 检查走后是否自将
    if (!isInCheck(board, color)) {
      valid.push([tr, tc]);
    }
    
    // 恢复棋盘
    board[r][c] = piece;
    board[tr][tc] = captured;
  }
  
  return valid;
}
```

## 六、ChessAI 详解

### 6.1 概述

基于 Minimax + Alpha-Beta 剪枝的象棋 AI，完全在前端运行。

### 6.2 API

```js
ChessAI.bestMove(board, color, difficulty, ChessRules)
// 返回：{ fr, fc, tr, tc } | null
```

### 6.3 难度参数

| 难度 | 搜索深度 | 位置评估表 | 延迟 |
|------|---------|-----------|------|
| easy | 2 | 否 | 300ms |
| normal | 3 | 否 | 500ms |
| hard | 5 | 是 | 800ms |

### 6.4 棋子价值

```js
const PIECE_VALUE = {
  K: 10000,  // 将/帅
  A: 120,    // 士/仕
  B: 150,    // 象/相
  R: 600,    // 車
  N: 300,    // 马
  C: 280,    // 炮
  P: 70,     // 兵/卒
};
```

### 6.5 位置评估表

仅 hard 难度使用，根据棋子在棋盘上的位置给予额外分数：
- 车：中路和肋道价值更高
- 马：靠近中心价值更高
- 炮：中炮位置价值高
- 兵：过河后越深入价值越高

### 6.6 算法流程

```
bestMove(board, color, difficulty, rules)
  │
  ├── 获取所有合法走法
  ├── 随机打乱（避免同局面永远走同一步）
  │
  └── 遍历每个走法
      ├── 模拟走棋
      ├── minimax(board, depth-1, -∞, +∞, !maximizing)
      ├── 恢复棋盘
      └── 更新最佳走法

minimax(board, depth, alpha, beta, maximizing)
  │
  ├── depth === 0 → evaluate(board)
  │
  ├── 获取当前方所有走法
  │   └── 无走法 → 返回极值（被将死）
  │
  └── 遍历走法（Alpha-Beta 剪枝）
      ├── 模拟走棋
      ├── 递归 minimax
      ├── 恢复棋盘
      └── 更新 alpha/beta，剪枝

evaluate(board, useTable)
  │
  └── 遍历所有棋子
      ├── 计算子力价值
      ├── 计算位置价值（若 useTable）
      └── 红方累加，黑方累减
```

### 7.2 渲染架构

- **Canvas 层**：绘制棋盘格线、九宫斜线、炮兵位标记、楚河汉界文字
  - 仅在 `_buildDOM()` 时绘制一次
  - 不响应交互（pointer-events 穿透）

- **棋子层**：每次 `_render()` 全量重建
  - 先渲染 `.move-hint`（落点提示）
  - 再渲染 `.chess-piece`（棋子）
  - 通过绝对定位覆盖在 Canvas 上方

- **坐标系统**：
  - `left = PADDING + col * CELL`
  - `top = PADDING + row * CELL`
  - 棋子使用 `transform: translate(-50%, -50%)` 居中

- **尺寸常量**：
  - `CELL = 54px`（格子大小）
  - `PADDING = 28px`（棋盘内边距）
  - 棋盘总宽：`54 * 8 + 28 * 2 = 488px`
  - 棋盘总高：`54 * 9 + 28 * 2 = 542px`

**注**：以上为桌面端固定尺寸。移动端采用响应式布局，动态计算格子大小（见第 11 章）。

## 九、常见问题定位

### 9.1 走棋相关

| 问题 | 可能原因 | 排查点 |
|------|---------|--------|
| 点击棋子无反应 | _canControl 返回 false | 检查 isSpectator、gameStarted、currentTurn、role |
| 走棋后对手看不到 | sendAction 未调用 | 检查 roomType 判断、sendAction 是否注入 |
| 走棋后棋盘错乱 | _applyMove 逻辑错误 | 检查 board 数组操作 |
| 将军检测失效 | isInCheck 逻辑错误 | 检查将位置查找、对方走法遍历 |

### 9.2 AI 相关

| 问题 | 可能原因 | 排查点 |
|------|---------|--------|
| AI 不走棋 | _triggerAI 未调用 | 检查 currentTurn !== role 判断 |
| AI 走棋很慢 | 搜索深度太大 | 检查 difficulty 参数 |
| AI 走出非法棋 | ChessRules 规则错误 | 检查对应棋子的 getCandidates |
| AI 永远走同一步 | 随机打乱失效 | 检查 shuffled 逻辑 |

### 9.3 状态同步相关

| 问题 | 可能原因 | 排查点 |
|------|---------|--------|
| 观战者棋盘不同步 | sync_state 未发送 | 检查 PvAI 模式 sendAction 调用 |
| 重连后棋盘空白 | game_state 未下发 | 检查服务端 record_game_state |
| 再来一局后状态残留 | reset 不彻底 | 检查 reset 方法所有状态清理 |

### 9.4 渲染相关

| 问题 | 可能原因 | 排查点 |
|------|---------|--------|
| 棋子位置偏移 | 坐标计算错误 | 检查 CELL、PADDING 常量 |
| 落点提示不显示 | getValidMoves 返回空 | 检查规则引擎 |
| 点击穿透失效 | pointer-events 设置错误 | 检查 CSS 层级 |
| 状态栏显示错误 | _updateStatusBar 逻辑 | 检查条件判断 |
| 移动端棋盘溢出 | 容器尺寸无效 | 检查 resize 监听和 calcBoardSize() |
| 棋子高光错位 | 伪元素使用固定像素 | 检查是否使用百分比定位 |

## 十、响应式布局规范（2026-03-19）

### 10.1 棋盘适配方案

**核心思路**：根据容器窄边动态计算格子大小，确保棋盘完整显示。

**JS 动态计算**：
```js
_calcBoardSize() {
  const COLS = 8, ROWS = 9;
  const RATIO = COLS / ROWS;
  
  // 获取容器可用空间，取窄边计算
  let availW = containerRect.width - margin * 2;
  let availH = containerRect.height - statusBarHeight - margin * 2;
  
  // 容器尺寸无效时，使用视口窄边计算
  if (availW <= 0 || availH <= 0) {
    availW = window.innerWidth - margin * 2;
    availH = window.innerHeight - statusBarHeight - margin * 2;
  }
  
  // 按象棋比例（8 列×9 行）计算格子大小
  const cellSize = Math.floor(Math.min(
    availW / COLS,
    availH / ROWS
  ));
  
  // 棋盘实际尺寸
  const boardWidth = cellSize * COLS;
  const boardHeight = cellSize * ROWS;
  
  // 同步更新 CSS 变量
  document.documentElement.style.setProperty('--piece-size', `${cellSize}px`);
}
```

**Resize 监听**：
```js
window.addEventListener('resize', () => {
  this._calcBoardSize();
});
```

### 10.2 棋子等比缩放

**规范**：
- 棋子直径 = 格子大小的 0.85 倍
- 使用 CSS 变量 `--piece-size` 统一控制
- 伪元素（高光、选中效果）使用百分比定位

**CSS 实现**：
```css
.chess-piece {
  width: calc(var(--piece-size) * 0.85);
  height: calc(var(--piece-size) * 0.85);
  /* 或者直接在 JS 中设置 style.width/height */
}

/* 高光使用百分比定位，跟随棋子大小缩放 */
.chess-piece::after {
  content: '';
  position: absolute;
  top: 12%;
  left: 18%;
  width: 60%;
  height: 30%;
  background: linear-gradient(180deg, rgba(255, 255, 255, 0.5) 0%, rgba(255, 255, 255, 0) 100%);
  border-radius: 50%;
}

/* 选中效果使用百分比 */
.chess-piece.selected::before {
  width: 90%;
  height: 90%;
}
```

### 10.3 横竖屏切换

**处理流程**：
1. 监听 `resize` 事件（横竖屏切换触发）
2. 重新调用 `_calcBoardSize()` 计算新尺寸
3. 更新容器 `style.width/height`
4. 同步更新 CSS 变量 `--piece-size`
5. 棋子自动跟随 CSS 变量缩放

**注意事项**：
- 容器高度需预留状态栏空间（约 40px）
- 确保棋盘始终居中显示
- 避免频繁重绘（可使用防抖）

### 10.4 CSS 变量优先级

**最佳实践**：
```js
// 同时在 documentElement 和 container 上设置，确保后备
document.documentElement.style.setProperty('--piece-size', `${size}px`);
this.container.style.setProperty('--piece-size', `${size}px`);
```

**CSS 中使用**：
```css
.chess-piece {
  width: calc(var(--piece-size, 54) * 0.85);  /* 54 为默认值 */
  height: calc(var(--piece-size, 54) * 0.85);
}
```
